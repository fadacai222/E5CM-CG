# E5CM-CG

> 一个用 `Python + Pygame + OpenCV` 重构街机游戏《e舞成名》流程与演出的项目。

这不是“做几个按钮、放几张图”的 UI demo。

这是一个明确朝着街机整机体验去推进的重构工程：从投币、1P/2P 选择、刷卡/游客进入、个人资料、大模式/子模式、选歌、加载、准备就绪、谱面播放到结算，整条主流程都已经被拆成独立场景，用 Python 重新搭起来了。

项目目标很直接：

- 复刻闭源街机游戏《e舞成名》的流程骨架、交互节奏、界面气质与厅机感
- 在保留原作味道的前提下，把资源组织、布局调试、渲染逻辑和场景切换做成可继续开发的工程
- 让这个项目不是一次性脚本，而是一套能继续迭代的 Python 重构版本

## 它现在已经有什么

- 完整主流程场景：
  `投币`、`玩家选择`、`登陆磁卡`、`个人资料`、`大模式`、`子模式`、`选歌`、`加载页`、`谱面播放器`、`结算`
- 开场动画与菜单背景视频：
  使用 `OpenCV` 处理开场视频与循环背景视频，和 `Pygame` 渲染链整合
- 街机式 UI 管线：
  全局点击特效、场景黑屏过渡、顶部栏、底部联网与信用值显示、全局 BGM 管理
- 谱面播放核心：
  独立谱面播放器场景、准备就绪动画、击中特效、判定/计分相关模块、血条与 HUD 渲染
- 数据与配置落地：
  `json/` 下有全局设置、加载页、选歌、个人资料布局等配置文件
- 个人资料系统：
  头像、昵称、段位、经验条、统计信息、软件信息页、二次弹窗交互
- 调试能力：
  所见即所得布局调试器、加载页调试器、准备就绪动画调试器、设置页预览调试器、选歌测试入口

## 这项目为什么值得看

因为它选的不是“最省事”的路线。

它没有停在静态原型，而是把街机项目最烦的那部分也做了出来：

- 多场景主循环和状态流转
- 视频背景与音频循环管理
- 以设计稿坐标为核心的布局映射
- 资源驱动的 UI 组织方式
- 谱面渲染器与游戏内 HUD
- 个人资料、信用值、模式选择等非游戏内系统
- 一堆为了继续开发而存在的调试器和测试入口

说白了，这仓库更像一台正在被 Python 重做的厅机，而不是一个“模仿风格”的小游戏外壳。

## 技术栈

- `Python 3`
- `Pygame`
- `OpenCV`
- `JSON` 配置驱动

当前代码明显偏向 Windows 本地开发与调试，目录结构、字体路径、资源组织也都是围绕这个环境写的。

## 运行

先准备依赖：

```bash
pip install pygame opencv-python
```

然后直接运行：

```bash
python main.py
```

如果你的环境里资源齐全，程序会从开场动画进入主流程，默认从投币场景开始。

## 仓库约定

当前仓库默认不纳入版本控制的内容：

- `songs/`
- `设计图/`
- Python 缓存与本地调试缓存

原因很简单：谱面歌曲、设计源文件和本地缓存会把仓库迅速撑爆，不适合直接跟源码一起走版本管理。
